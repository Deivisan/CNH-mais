# GSD - Get Shit Done

## Sobre
Sistema de meta-prompting e context engineering para desenvolvimento orientado a especificação.

## Comandos Principais

### `/gsd:new-project`
Inicializa novo projeto com Questions → Research → Requirements → Roadmap

### `/gsd:discuss-phase [N]`
Captura decisões de implementação antes do planejamento

### `/gsd:plan-phase [N]`
Pesquisa + plano + verificação para uma fase

### `/gsd:execute-phase [N]`
Executa todos os planos em ondas paralelas

### `/gsd:verify-work [N]`
Teste de aceitação do usuário

### `/gsd:quick`
Tarefas ad-hoc sem planejamento completo

## Workflow
1. Initialize → 2. Discuss → 3. Plan → 4. Execute → 5. Verify → 6. Ship

## Princípios
- Context engineering por trás das cenas
- Commits atômicos por tarefa
- Execução em ondas paralelas
- Verificação integrada

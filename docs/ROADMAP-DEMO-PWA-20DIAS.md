# roadmap demo pwa - 20 dias

## objetivo

evoluir a experiência web appificada (demo pwa) para uma representação fiel do produto, com foco em fluxo completo e consistência de regras.

## trilha de execução

### dias 1-5 — fundação e cobertura estrutural
- [x] criar estrutura `demo-pwa/` (pwa estático)
- [x] manter landing principal em paralelo (sem quebra)
- [x] implementar navegação por painéis (menu lateral + atalhos)
- [x] consolidar login/cadastro de 3 perfis emulado

### dias 6-10 — cobertura funcional principal
- [x] fluxo candidato completo (onboarding, pagamento, match, aulas, chat, mapas)
- [x] perfil público do instrutor (8 seções)
- [x] painel do instrutor com abas essenciais + extras operacionais
- [x] backoffice administrativo com visão executiva

### dias 11-15 — robustez operacional
- [x] bloco admin “tela 4 gerenciamento do instrutor” com 12 blocos
- [x] emulação de disputa/auditoria e ações administrativas
- [ ] aumentar fidelidade da agenda com slot-level de veículo + bloqueios avançados
- [ ] reforçar estados de erro e feedbacks transacionais

### dias 16-20 — acabamento e entrega
- [ ] revisão visual final (spacing, tipografia, microinterações)
- [ ] revisão de copy e consistência semântica
- [ ] validação completa de navegação e roteiros de demonstração
- [ ] documentação final de handoff

## critérios de pronto

- navegação completa sem links quebrados
- cobertura de telas-chave dos 3 perfis
- regras principais do domínio representadas visualmente
- documentação com status claro: validado x pendente
